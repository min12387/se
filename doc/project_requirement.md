본 문서는 Mini Drive 시스템 개발에 필요한 요구사항을 체계적으로 정의하기 위해 작성되었다.

## 1.2 문서 범위
본 문서는 Mini Drive 시스템의 전반적인 요구사항을 다루며 사용자 기능, 관리자 기능, 성능, 보안, 인터페이스 및 품질 요구사항을 포함한다.

---

# 2. 시스템 개요

## 2.1 시스템 정의
Mini Drive는 조직 내 파일을 중앙 서버에서 관리하고, 사용자 간 파일 공유 및 협업을 지원하는 클라우드 기반 파일 관리 플랫폼이다.

---

## 2.2 주요 기능

| 기능 코드 | 기능 설명 |
|-----------|----------|
| MD-01 | 다양한 형식의 파일 업로드 및 다운로드 |
| MD-02 | 계층형 폴더 구조 생성 및 관리 |
| MD-03 | 사용자 및 그룹 기반 파일 공유 |
| MD-04 | 파일 검색 및 필터링 기능 |
| MD-05 | 파일 변경 이력 관리 및 복구 |

---

## 2.3 용어 설명

| 용어 | 의미 |
|------|------|
| 접근 권한 | 사용자에게 허용된 파일 접근 범위 |
| 공유 링크 | 외부 사용자 접근을 위한 URL |
| 버전 관리 | 파일 변경 내역을 저장하고 이전 상태로 복구하는 기능 |

---
