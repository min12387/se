# 1. 서론

## 1.1 문서 목적 및 범위

본 문서는 Mini Drive 시스템에 대한 테스트를 수행하고 그 결과를 기록하기 위해 작성되었다.

테스트를 통해 요구사항 정의서에서 정의한 기능 요구사항(FR) 및 비기능 요구사항(NFR)의 충족 여부를 검증하며, 시스템의 품질 수준을 평가한다.

본 문서의 범위는 다음과 같다.

* 기능 요구사항 검증
* 비기능 요구사항 검증
* 성능 및 보안 테스트
* 브라우저 호환성 테스트
* 테스트 결과 분석

## 1.2 프로젝트 개요

Mini Drive는 조직 내 파일을 중앙에서 관리하고 사용자 간 파일 공유 및 협업을 지원하는 클라우드 기반 파일 관리 시스템이다.

주요 기능은 다음과 같다.

* 파일 업로드 및 다운로드
* 폴더 생성 및 관리
* 파일 공유 및 권한 설정
* 공유 링크 생성
* 파일 검색 및 필터링
* 파일 버전 관리
* 삭제 파일 복구

## 1.3 용어 정의

| 용어         | 설명                                     |
| ---------- | -------------------------------------- |
| Mini Drive | 본 프로젝트에서 개발하는 클라우드 파일 관리 시스템           |
| 공유 링크      | 외부 사용자가 파일에 접근할 수 있도록 생성되는 URL         |
| RBAC       | 역할 기반 접근 제어(Role-Based Access Control) |
| 버전 관리      | 파일 변경 이력을 저장하고 이전 상태로 복구하는 기능          |

## 1.4 참조 문서

* 프로젝트 정의서
* 프로젝트 관리 계획서
* 요구사항 정의서
* 요구사항 분석서

# 2. 테스트 개요

## 2.1 테스트 범위

본 테스트는 블랙박스 테스트를 기반으로 수행하였다.

| 구분     | 내용               |
| ------ | ---------------- |
| 사용자 관리 | 로그인 기능 검증        |
| 파일 관리  | 업로드, 다운로드, 삭제    |
| 공유 기능  | 공유 링크 생성 및 권한 관리 |
| 검색 기능  | 파일 검색 및 필터링      |
| 버전 관리  | 이전 버전 복구         |
| 보안     | 접근 권한 검증         |
| 성능     | 응답 시간 및 동시 접속 처리 |

## 2.2 테스트 항목 및 통과 기준

| 항목      | 통과 기준           |
| ------- | --------------- |
| 로그인     | 정상 계정 로그인 성공    |
| 파일 업로드  | 지원 파일 형식 업로드 성공 |
| 파일 다운로드 | 업로드된 파일 다운로드 성공 |
| 파일 공유   | 권한 설정 후 공유 성공   |

| 보안 테스트  | 비인가 접근 차단       |
