# 1. 서론

## 1.1 문서 목적 및 범위

본 문서는 Mini Drive 시스템에 대한 테스트를 수행하고 그 결과를 기록하기 위해 작성되었다.

테스트를 통해 요구사항 정의서에서 정의한 기능 요구사항(FR) 및 비기능 요구사항(NFR)의 충족 여부를 검증하며, 시스템의 품질 수준을 평가한다.

본 문서의 범위는 다음과 같다.

* 기능 요구사항 검증
* 비기능 요구사항 검증
* 성능 및 보안 테스트
* 브라우저 호환성 테스트
* 테스트 결과 분석

## 1.2 프로젝트 개요

Mini Drive는 조직 내 파일을 중앙에서 관리하고 사용자 간 파일 공유 및 협업을 지원하는 클라우드 기반 파일 관리 시스템이다.

주요 기능은 다음과 같다.

* 파일 업로드 및 다운로드
* 폴더 생성 및 관리
* 파일 공유 및 권한 설정
* 공유 링크 생성
* 파일 검색 및 필터링
* 파일 버전 관리
* 삭제 파일 복구

## 1.3 용어 정의

| 용어         | 설명                                     |
| ---------- | -------------------------------------- |
| Mini Drive | 본 프로젝트에서 개발하는 클라우드 파일 관리 시스템           |
| 공유 링크      | 외부 사용자가 파일에 접근할 수 있도록 생성되는 URL         |
| RBAC       | 역할 기반 접근 제어(Role-Based Access Control) |
| 버전 관리      | 파일 변경 이력을 저장하고 이전 상태로 복구하는 기능          |

## 1.4 참조 문서

* 프로젝트 정의서
* 프로젝트 관리 계획서
* 요구사항 정의서
* 요구사항 분석서

# 7. 결론

본 프로젝트에 대한 테스트 결과, 요구사항 정의서에서 정의한 기능 요구사항(FR) 및 비기능 요구사항(NFR)을 모두 만족하였다.

파일 업로드, 다운로드, 공유, 검색 및 버전 복구 기능이 정상적으로 수행되었으며, 성능 테스트와 보안 테스트 또한 기준을 충족하였다.

또한 SSL/TLS 기반 암호화 통신과 권한 기반 접근 제어(RBAC)가 정상적으로 동작함을 확인하였다.

따라서 Mini Drive 시스템은 운영 환경에 배포 가능한 수준의 품질을 확보한 것으로 판단한다.
